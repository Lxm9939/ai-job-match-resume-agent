# AI秋招岗位匹配与简历优化助手｜AI产品原型与Agent应用实践

## 项目背景

秋招投递中，候选人需要快速理解大量岗位 JD，并判断简历是否真正匹配岗位要求。JD 中的职责、硬技能、业务关键词和隐含能力常常混在一起，简历中已有经历也未必被表达成招聘方能识别的证据点，导致关键词覆盖不足、项目亮点不突出、投递话术重复手写。基于这些痛点，我设计并实现了一个网页版 MVP：AI 秋招岗位匹配与简历优化助手。

## 我的角色

我负责产品原型设计与完整实现，覆盖需求分析、用户流程、功能拆解、Agent 工作流、交互页面、规则评测、测试用例和 GitHub 展示文档。项目从求职者真实场景出发，将“上传简历、输入 JD、获得匹配分析和投递建议”设计成一条清晰流程，并沉淀 README、项目计划书、工作流文档、测试用例和 Agent 输出样例。

## 核心功能

当前版本已实现 Streamlit 网页端，支持简历文件上传和文本粘贴，文件类型包括 `.txt`、`.pdf`、`.docx`。用户粘贴岗位 JD 并填写目标岗位方向后，系统输出 7 个 tab：JD 解析、证据匹配、关键词覆盖、匹配评分、简历优化、投递话术和最终报告。报告支持 Markdown 下载和 Word docx 导出。项目提供 Mock 模式，没有 OpenAI API Key 也可本地演示。

## 产品设计思路

我没有把它设计成简单的“AI 改简历”工具，而是拆成“岗位理解、简历解析、证据匹配、差距定位、表达优化、投递沟通”六个步骤。真实求职中，关键不是生成更漂亮的文字，而是知道岗位需要什么、已有经历能证明什么、缺失在哪里，以及如何在不编造经历的前提下优化表达。因此页面采用结构化表格和 Markdown 报告，方便复盘和解释产品逻辑。

## Agent 工作流

项目采用 8 个 Agent 模块：JD Parser 解析岗位职责、技能、工具栈和隐含能力；Resume Parser 提取教育、项目、实习、技能和成果指标；Evidence Mapping 将岗位要求映射到简历证据；Keyword Coverage 分析关键词覆盖；Scoring 按技能、项目、关键词、职责、教育背景生成可解释评分；Resume Optimizer 给出改写建议；Outreach 生成多渠道话术；Report 汇总最终报告。每个 Agent 输入输出相对独立，便于调试、测试和后续替换模型。

## 技术实现

技术栈使用 Python、Streamlit、Pydantic、python-docx、pypdf 和 OpenAI API 封装。`src/llm_client.py` 统一管理 OpenAI 与 Mock 模式，避免 API Key 写入代码；`.env.example` 提供环境变量模板。文档解析模块支持多格式简历输入，工作流模块串联 8 个 Agent，报告导出模块生成 Word 报告。项目配置 pytest 和 GitHub Actions CI，在 push 和 pull request 时自动运行测试。

## 评测与优化

我基于示例简历和示例 JD 设计测试，覆盖 mock 工作流、文档解析、文本清洗、关键词提取、去重逻辑和报告导出。当前本地 pytest 已通过，并完成 Streamlit 启动验证。为提升作品集展示效果，我优化了关键词去重，保持原有顺序，避免报告重复堆叠；同时补充截图说明、Agent 输出样例和项目计划书，增强可读性和面试讲解价值。
