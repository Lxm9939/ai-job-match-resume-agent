# AI 岗位搜索、匹配评分与求职优化助手：项目计划书

## 项目概述

本项目是一个面向不同求职阶段候选人的网页版求职辅助工具。V1 支持单个 JD 深度分析；V2 支持导入多个岗位并生成匹配排序；V3 支持从合规公开岗位源抓取职位后进入 V2；V3.2 进一步提供岗位匹配数据分析 Dashboard。各版本共用 Agent 与结构化数据模型。

适用于实习、秋招、校招、社招、转行、海外留学生回国求职和在职跳槽等场景。

MVP 使用 Python + Streamlit 实现，默认支持 mock/规则模式，因此没有 API Key 也可以本地运行和作品集演示；当配置 OpenAI API Key 后，可以切换为 LLM 增强模式。

## 背景与痛点

岗位申请通常不是“写一份简历投所有岗位”，而是需要针对不同 JD 快速判断匹配度并调整表达。典型痛点包括：

- JD 长且复杂，岗位职责、任职要求、业务场景和加分项混在一起。
- 简历经历与岗位要求之间缺少明确证据映射。
- 关键词覆盖不足，导致初筛系统或招聘方快速浏览时无法识别匹配点。
- 候选人难以判断当前简历更适合哪个岗位方向。
- 批量投递时，邮件、Boss 直聘、LinkedIn 和内推话术需要重复手写，效率低。
- LLM 直接改简历容易出现编造经历风险，需要明确约束。

## 目标用户

- 应届生和实习求职者。
- 海外留学生及回国求职用户。
- 社招候选人与在职跳槽用户。
- 转行到数据、AI、产品和分析岗位的人。
- 申请 AI 应用、Data Agent、数据分析、商业分析、BI 分析和产品经理等岗位的人。
- 需要批量比较岗位、优化简历和准备面试的求职者。

## 核心功能

1. 简历输入：支持文本粘贴和 `.txt` / `.pdf` / `.docx` 上传。
2. 求职偏好：支持求职阶段、目标方向、岗位类型和城市/地区偏好。
3. JD 输入：支持岗位 JD 粘贴和目标岗位方向填写。
4. JD 解析：抽取岗位名称、公司、地点、职责、技能、工具、业务关键词和隐含能力。
5. 简历解析：抽取教育、项目、实习、技能、指标、关键词和可迁移能力。
6. 证据匹配：输出“岗位要求 - 简历证据 - 证据强度 - 建议补充表达”。
7. 关键词覆盖：输出已覆盖、弱覆盖和未覆盖关键词。
8. 匹配评分：按固定权重生成可解释 0-100 分。
9. 简历优化：基于原始经历给出 bullet 改写建议，不编造内容。
10. 投递话术：生成 Boss、邮件、LinkedIn、内推和面试自我介绍。
11. 报告导出：支持 Markdown 和 Word `.docx` 报告。
12. 批量岗位导入：支持 CSV / Excel 岗位列表和 `---JOB---` 分隔的多 JD 文本。
13. 岗位推荐排序：复用 V1 五维评分，对多个岗位按匹配总分降序排列并给出投递分档。
14. 面试准备：为每个岗位生成可能问题、项目讲解重点、技术/业务准备、风险追问和回答策略。
15. 批量报告：下载包含排行榜和逐岗位详情的 Markdown 报告。
16. 公开岗位抓取：检查 robots.txt 后访问配置的公开 Careers 页面，筛选并保留来源链接。
17. 抓取 Demo：使用本地示例数据演示 V3，不依赖网站可用性。
18. 数据分析 Dashboard：展示岗位分布、推荐结论、质量分布和 Top 关键词。

## Agent 工作流

项目采用模块化 Agent 编排：

1. Input Router 接收简历文本、上传文件解析结果、JD 和目标岗位方向。
2. JD 解析 Agent 将非结构化 JD 转为 `JDAnalysis`。
3. 简历解析 Agent 将非结构化简历转为 `ResumeAnalysis`。
4. 简历证据提取 Agent 映射 JD 要求和简历证据。
5. 关键词覆盖 Agent 计算关键词覆盖情况。
6. 匹配评分 Agent 按权重计算总分和分项原因。
7. 简历优化 Agent 输出 bullet 改写建议。
8. 话术生成 Agent 输出多渠道投递表达。
9. 最终报告 Agent 汇总为 Markdown 报告。
10. 报告导出模块将结构化结果转换为 Word 文档。

V2 在 V1 之上新增：

1. 岗位列表解析 Agent 将 CSV / Excel / 多 JD 文本转为 `List[JobPosting]`。
2. 批量匹配 Agent 对每个岗位复用 V1 的解析、证据、关键词、评分、优化和话术 Agent。
3. 面试准备 Agent 基于真实简历证据生成逐岗位准备清单。
4. 批量工作流按总分排序，汇总为 `BatchMatchResult` 和 Markdown 报告。

## 技术架构

- 前端和交互：Streamlit。
- 后端语言：Python。
- 数据模型：Pydantic。
- 文档解析：`pypdf`、`python-docx`、文本解码。
- 表格导入：`pandas`、`openpyxl`。
- 公开页面解析：`requests`、`BeautifulSoup`、`urllib.robotparser`。
- LLM 适配：OpenAI API 封装在 `src/llm_client.py`。
- Mock 模式：本地规则、关键词库和启发式评分。
- 报告导出：Markdown 原文下载和 `python-docx` 生成 Word 报告。
- 测试：pytest。

## 数据输入输出

输入：

- 简历文件：`.txt`、`.pdf`、`.docx`。
- 简历文本：用户直接粘贴。
- 岗位 JD：用户粘贴。
- 目标岗位方向：例如 AI 产品经理、数据分析师、商业分析师、数据产品经理、AI Agent 产品经理。
- 批量岗位：CSV / Excel 文件或使用 `---JOB---` 分隔的多 JD 文本。
- 求职偏好：目标岗位方向、城市、岗位类型和公司偏好。
- 公开岗位源：用户维护的 JSON 配置，或本地示例抓取结果。

输出：

- JD 解析结果。
- 简历证据匹配表。
- 关键词覆盖分析。
- 匹配评分与解释。
- 简历优化建议。
- 投递话术。
- 最终 Markdown 报告。
- Word `.docx` 报告。
- 批量岗位匹配排行榜。
- 每个岗位的面试准备清单。
- 批量 Markdown 报告。
- 抓取岗位 CSV、来源状态和可核查的岗位链接。

## MVP 范围

MVP 优先保证可运行、可演示、可上传 GitHub：

- 单个简历与单个 JD 的匹配分析，以及一份简历对多个导入岗位的批量分析。
- 本地 mock 模式完整闭环。
- OpenAI API 可选增强。
- Markdown/表格形式展示，不追求复杂图表。
- V1 Word 报告导出、V2 Markdown 报告导出。
- 基础 pytest 自动测试。

暂不包含：

- 简历版本管理。
- 在线用户账号。
- 数据库持久化。
- PDF 导出和复杂排版。
- 招聘平台自动爬取。
- 批量 Word 报告导出。

## V2 版本规划与实现状态

V2 的目标是把 V1 的“单岗位深度分析”扩展为“多岗位决策支持”，同时不改变 V1 主流程。

已实现：

- 侧边栏切换 V1 单 JD 与 V2 批量推荐。
- CSV / XLSX 岗位列表导入与多 JD 文本拆分。
- `JobPosting`、`JobMatchResult`、`BatchMatchResult`、`InterviewPrep` 等 Pydantic 模型。
- 岗位列表解析、批量匹配、面试准备三个新增 Agent。
- 岗位排行榜、逐岗位详情、简历优化、面试准备与投递话术。
- 批量 Markdown 报告和 5 个示例岗位。
- 岗位解析与批量工作流自动测试。

后续规划：

- 批量 Word 报告。
- 多版本简历对比。
- 在线部署和 UI 自动化测试。

## V3 公开岗位源抓取

V3 的目标是在不接触登录平台和反爬绕过的前提下，将公开岗位获取接到 V2 批量匹配上游。用户可以配置公司官网 Careers 页面，系统检查 robots.txt 后低频访问、解析岗位、按偏好筛选，再复用 V2 输出排序、简历优化、面试准备和投递话术。

当前已实现：

- Streamlit 第三种模式“公开岗位源自动抓取”。
- JSON 岗位源配置上传和默认示例配置。
- `JobSource`、`CrawledJob`、`CrawlResult`、`JobSearchPreference` 等结构化模型。
- robots.txt 检查，网络失败时默认跳过。
- 公开静态 HTML 的通用岗位链接与文本片段解析。
- 明确 User-Agent、请求超时、5 MB 页面上限、至少 1 秒来源间隔、最大岗位数和一小时缓存。
- 岗位方向、关键词、城市和岗位类型筛选。
- 抓取岗位 CSV 和匹配 Markdown 报告下载。
- 本地示例抓取结果，可在没有真实公开来源时完整演示。

合规边界：

- 只支持公开可访问、无需登录且 robots.txt 允许的公司官网 Careers 或静态 HTML 页面。
- 不支持 Boss 直聘、智联招聘、猎聘、前程无忧、拉勾等登录或强反爬平台。
- 不处理验证码，不模拟登录，不使用 Cookie 绕过、代理池或浏览器指纹绕过。
- robots.txt 无法检查、明确拒绝或异常响应时安全跳过。
- 抓取结果始终保留来源链接，供用户核查。

当前局限：

- 通用解析器不能保证适配所有网页结构，动态渲染页面可能无法提取。
- RSS/API 和站点专属字段适配器尚未实现。
- V3 结果用于求职信息整理，不代表招聘网站官方推荐。

明确说明：当前 V3 支持公开岗位源抓取和示例抓取结果演示，不支持登录平台、验证码平台和反爬绕过。

## V3.1 稳定增强

V3.1 不扩展新的岗位来源类型，重点提升已有抓取结果的可靠性和可解释性。

已实现：

- 按来源 URL 去重；链接缺失时按公司、岗位名称和城市去重。
- 重复组保留字段更完整、JD 更长的版本。
- 记录去重前、去重后、重复记录和重复组数量。
- 按岗位名称、公司、城市、JD 长度、技能关键词、来源链接和发布日期生成 0-100 质量分。
- 输出高/中/低质量标签和字段 warning。
- 低质量岗位不直接删除，在页面和详情中标记为“低置信度”。
- 展示原始、去重、筛选、质量分布、robots 跳过和抓取失败统计。

产品解释：

- 匹配评分衡量“候选人与岗位的适配程度”。
- 岗位质量评分衡量“抓取记录的字段完整度”。
- 两个分数保持独立，避免用户把低质量抓取数据误认为低匹配岗位。

## V3.2 数据分析 Dashboard

V3.2 将项目从逐岗位 AI 分析进一步扩展为“AI Agent + 数据分析 Dashboard”。该版本不增加新的抓取或业务流程，只对现有 V2/V3 结果做轻量聚合。

已实现：

- 岗位总数、平均/最高/最低匹配分。
- 推荐、谨慎和不优先投递数量。
- 高/中/低质量与低置信度岗位数量。
- 城市、岗位类型、推荐结论和岗位质量分布图。
- 高匹配岗位、缺失关键词、常见技能和风险关键词 Top 10。
- 排行榜 CSV、缺失关键词 CSV 和分析摘要 Markdown。
- V2/V3 共用 Analytics 模块与 Dashboard 组件。

作品集价值：

- 展示从非结构化 JD 到结构化数据、Agent 分析、指标聚合和可视化的完整链路。
- 体现指标定义、数据清洗、频次统计、Top N 分析和结果导出能力。
- 面试时可以分别讲 Agent 模块化、启发式评分、数据质量、Dashboard 指标和合规边界。

## 测试方案

- 单元/集成测试：`pytest` 验证 V1/V2/V3 mock 工作流、Word 导出、岗位解析、排序、面试准备、robots 检查、岗位去重、质量评分和 Dashboard 聚合指标。
- 手动测试：覆盖文本输入、文件上传、缺失输入、不同岗位 JD、OpenAI/mock 模式切换。
- UI 验证：启动 Streamlit 并访问本地地址，确认页面返回 HTTP 200。
- 安全检查：确认 `.env` 被忽略，`.env.example` 不含真实 Key。

## 风险与解决方案

| 风险 | 影响 | 解决方案 |
| --- | --- | --- |
| LLM 输出 JSON 不稳定 | 结构化解析失败 | 统一封装 `LLMClient`，失败时回退本地规则 |
| 无 API Key 无法演示 | 作品集体验受限 | 默认 mock 模式完整运行 |
| 简历优化出现编造 | 求职风险高 | Agent 提示词和 UI 明确“不编造”原则 |
| PDF/DOCX 解析质量不稳定 | 简历文本可能缺失 | 支持粘贴文本作为兜底 |
| 评分过度精确 | 用户误解结果 | 输出分项原因和风险提示 |
| 关键词库不够完整 | 覆盖分析偏粗 | 后续增加岗位方向专属关键词库 |

## 项目亮点

- 把求职简历优化拆成可解释的多 Agent 工作流。

- 明确防止 LLM 编造经历，符合真实求职场景。
